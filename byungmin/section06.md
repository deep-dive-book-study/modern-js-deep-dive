## 6장 데이터 타입

### 숫자 타입
1. 하나의 숫자 타입
- 자바스크립트는 int, long, float, double 등 정수와 실수를 구분하는 다양한 타입을 제공하지 않는다.
- 모든 숫자를 64비트 부동소수점 형식의 실수로만 처리함
- 즉, 겉보기에는 정수여도 내부적으로는 모두 실수로 저장됨
```javascript
var integer = 10;
var double = 10.0;

console.log(integer === double); // true (둘 다 실수 10으로 처리됨)
```

### undefined / null 타입
1. undefined (시스템적 비어 있음)
- 주체: 자바스크립트 엔진이 할당
- 의미: 변수는 선언되었으나 아직 값이 할당되지 않은 상태
- 발생 상황:
    - 변수 선언 후 초기화되지 않았을 때 (var는 런타임 이전, let/const는 선언문 도달 시)
    - 존재하지 않는 객체 프로퍼티에 접근할 때
    - 함수에서 매개변수를 전달받지 못했거나 return 문이 없을 때

2. null (의도적 비어 있음)
- 주체: 개발자가 직접 할당
- 의미: 변수에 값이 없음을 명시하거나, 이전에 참조하던 값을 의도적으로 제거했을 때 사용

### 데이터 타입이 필요한 이유
- 값의 종류에 따른 메모리 공간 사이즈 확보
- 값을 읽어올 때 메모리 주소의 참조 범위 결정
- 2진수 데이터의 해석 방식 결정

### 동적 타이핑
- 자바스크립트 변수는 값의 할당에 의해 타입이 결정(타입추론)된다. 그리고 재할당에 의해 변수의 타입은 언제든지 동적으로 변할 수 있다.
- 현재 변수에 할당되어 있는 값에 의해 변수의 타입이 동적으로 결정된다.
