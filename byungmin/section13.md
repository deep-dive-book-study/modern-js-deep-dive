## 13장 스코프

### 스코프 정의 (Scope)
1. 정의
- 식별자가 유효한 범위: 모든 식별자는 자신이 선언된 위치에 따라 다른 코드가 자신을 참조할 수 있는 범위가 결정되는데, 이를 스코프라고 한다.
- 식별자 결정(Identifier Resolution): 자바스크립트 엔진이 이름이 같은 두 변수 중 어떤 변수를 참조해야 할지 결정하는 규칙

2. 스코프의 필요성 (이름 충돌 방지)
- 만약 스코프라는 개념이 없다면, 프로그램 전체에서 똑같은 이름의 변수는 단 하나만 사용해야 한다.
- 스코프를 통해 네임스페이스를 구분함으로써, 다른 스코프(범위) 내에서는 같은 이름의 식별자를 사용할 수 있게 한다.
```javascript
var x = 'global'; // 전역 스코프

function foo() {
  var x = 'local'; // foo 함수 스코프
  console.log(x);  // 'local' 출력 (가까운 스코프의 변수를 참조함)
}

foo();

console.log(x);    // 'global' 출력
```

### 스코프 체인 (Scope Chain)
- 정의: 스코프가 함수의 중첩에 의해 계층적인 구조를 갖는 것
- 동작: 엔진은 변수를 참조할 때 현재 스코프에서 시작하여 상위 스코프 방향으로 이동하며 선언된 식별자를 검색
- 결과: 상위 스코프에서 선언한 변수를 하위 스코프에서 참조할 수 있지만, 하위 스코프에서 선언한 변수를 상위 스코프에서 참조할 수는 없음 (단방향성)

### 렉시컬 스코프 (정적 스코프)
1. 정의
- 정의: 함수를 어디서 호출했는지가 아니라, 함수를 어디서 정의했는지에 따라 상위 스코프를 결정하는 규칙을 의미한다.
- 자바스크립트를 포함한 대부분의 현대 프로그래밍 언어는 이 방식을 따르며, 이를 `정적 스코프(Static Scope)`라고도 부른다.

2. 작동 메커니즘: 상위 스코프의 결정과 참조
- 정의 시점의 환경 저장: 함수 정의가 평가되어 함수 객체가 생성될 때, 엔진은 함수가 정의된 위치의 실행 환경(Lexical Environment)을 확인한다.
- 내부 슬롯([[Environment]])에 기록: 생성된 함수 객체는 자신의 내부 슬롯인 [[Environment]]에 상위 스코프의 참조를 저장한다. 이는 함수가 존재하는 한 영구적으로 유지된다.
- 식별자 검색(Identifier Resolution): 함수가 호출되면, 함수는 호출된 위치와 관계없이 자신이 저장한 [[Environment]] 참조를 통해 상위 스코프 체인을 따라가며 변수를 검색한다.
```javascript
// 런탕임 이전 단계
var x; // 1. 식별자 등록 및 undefined 초기화
function foo() { ... } // 2. 식별자 등록 및 함수 객체 할당 (바디 포함)
function bar() { ... } // 3. 식별자 등록 및 함수 객체 할당 (상위 스코프를 전역으로 고정)

// 런타임 단계
var x = 1; // 전역 변수 x에 1을 할당한다.

function foo() {
    var x = 10; // foo 지역 변수 x에 10을 할당한다.

    bar();
}

function bar() {
    console.log(x); // bar의 상위 스코프인 전역의 x를 참조한다.
}

foo(); // 출력: 1 (bar 실행 시 전역 스코프의 x 참조)
bar(); // 출력: 1 (bar 실행 시 전역 스코프의 x 참조)
```
