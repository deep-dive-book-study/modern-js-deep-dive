## 14장 전역 변수의 문제점

### 지역 변수의 생명 주기

1. 생명 주기의 시작: 함수 호출

- 지역 변수는 전역 변수처럼 런타임 이전(애플리케이션 시작 시점)에 생성되지 않는다.
- 생성 시점: **함수가 호출되어 함수 몸체의 코드가 실행되기 직전**, 자바스크립트 엔진에 의해 해당 함수의 스코프가 생성되면서 변수들이 등록된다.
- 호이스팅 범위: 지역 변수의 호이스팅은 해당 함수 내부에서만 발생한다. 즉, 함수 내부의 변수 선언문이 함수 몸체의 최상단으로 끌어올려진 것처럼 동작한다.

2. 생명 주기의 종료: 함수 종료

- 소멸 시점: 일반적으로 함수가 실행을 마치고 종료되면 함수가 생성한 스코프도 소멸하며, 그 스코프에 속했던 지역 변수도 메모리에서 해제된다.
- 예외 상황 (클로저): 만약 누군가 함수의 스코프를 여전히 참조하고 있다면, 함수가 종료되어도 스코프와 변수는 소멸하지 않고 유지된다.

| 구분      | 전연 변수 (Global Variable) | 지역 변수 (Local Variable) |
|---------|---------------------|----------------------|
| 생성 시점   | 애플리케이션 시작 시 (런타임 이전) | 함수 호출 시 |
| 소멸 시점   | 애플리케이션 종료 시 | 함수 종료 시 |
| 호이스팅 볌위 | 전역 코드 전체의 최상단 | 함수 내부의 최상단 |
| 메모리 점유  | 프로그램 종료 시까지 계속 점유 | 함수 실행 동안에만 점유 |

```javascript
var x = 'global'; // 전역 변수 x 생성 (애플리케이션 종료 시까지 생존)

function foo() {
    // [foo 호출 시점] 지역 변수 y 생성 및 undefined 초기화 (지역 호이스팅)
    console.log(y); // undefined
    
    var y = 'local'; // 값 할당
    console.log(y); // 'local'
    
    return y;
    // [foo 종료 시점] 지역 변수 y 소멸
}

foo();
console.log(y); // ReferenceError: y is not defined (y는 이미 소멸함)
```

### 전역 변수의 문제점
1. 암묵적 결합 (Implicit Coupling)
- 전역 변수는 코드 어디서든 참조하고 변경할 수 있다. 이는 모든 코드가 전역 변수를 공유하는 암묵적 결합을 허용하는 것이다.
- 위험성: 변수의 유효 범위가 클수록 코드의 가독성은 나빠지고, 의도치 않게 상태가 변경될 위험이 높아진다.

2. 긴 생명 주기 (Long Lifecycle)
- 전역 변수는 애플리케이션이 실행될 때 생성되어 종료될 때까지 메모리에 잔존한다.
- 위험성: 메모리 리소스를 오랜 기간 소비하며, 특히 var 키워드는 중복 선언을 허용하므로 변수 이름이 겹칠 경우 의도치 않은 재할당이 발생할 가능성이 매우 크다.

3. 스코프 체인 상에서 종점에 존재
- 전역 변수는 스코프 체인의 가장 바깥쪽에 위치한다.
- 위험성: 자바스크립트 엔진이 변수를 찾을 때 가장 마지막에 검색하게 된다. 즉, 변수 검색 속도가 가장 느리다. (차이는 미미하지만 성능에 영향을 줄 수 있다.)

4. 네임스페이스 오염
- 자바스크립트는 파일이 분리되어 있어도 하나의 전역 스코프를 공유하는 경우가 많다.
- 위험성: 다른 파일에서 동일한 이름으로 명명된 전역 변수나 전역 함수가 같은 스코프 내에 존재할 경우, 예상치 못한 결과를 초래한다.

### 전역 변수 사용을 억제하는 방법
전역 변수를 반드시 써야 할 이유가 없다면 지역 변수를 사용하는 것이 원칙이다. 이를 위해 다음과 같은 방법을 사용한다.
- 즉시 실행 함수 (IIFE): 모든 코드를 즉시 실행 함수로 감싸서 지역 변수로 만든다.
- 네임스페이스 객체: 전역에 객체를 하나 만들고, 관리할 변수를 그 객체의 프로퍼티로 추가한다. (충돌은 줄지만 객체 자체는 전역 변수다.)
- 모듈 패턴: 클로저를 활용하여 관련 있는 변수와 함수를 하나로 묶어 은닉한다.
- ES6 모듈: import와 export를 사용하면 파일 자체가 독자적인 모듈 스코프를 갖게 되어 전역 변수 오염을 원천 차단한다.
