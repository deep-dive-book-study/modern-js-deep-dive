### 4장 변수

#### 변수

하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름.
기억하고 싶은 값을 메모리에 저장하고, 저장된 값을 읽어들여 재사용하기 위해 사용

#### 식별자

어떤 값을 구별해서 식별할 수 있는 고유한 이름
값이 저장되어 있는 **메모리 주소**와 매핑 관계(값 자제 저장 X)

#### 변수 선언

var, let, const로 메모리 공간 확보(let, const는 ES6에서 도입)

1. 선언 단계: 변수 이름을 등록해서 자바스크립트 엔진에 변수의 존재를 알린다
2. 초기화 단계: 값을 저장하기 위한 메모리 공간을 확보하고 암묵적으로 undefined 할당 후 초기화
   (변수 이름을 비롯한 모든 식별자는 실행 컨텍스트에 등록)

#### 변수 선언의 실행 시점과 변수 호이스팅

변수 선언이 소스코드의 어디에 있든 상관없이 다른 코드보다 먼저 실행(런타임 이전 단계에서 실행)
**-> 변수 호이스팅**

#### 값의 할당

자바스크립트 엔진은 변수 선언과 값의 할당을 하나의 문으로 단축 표현해도 변수 선언과 값의 할당을 2개의 문으로 나누어 각각 실행

- 변수 선언: 런타임 이전에 먼저 실행(undefined가 할당)
- 값의 할당: 런타임에 실행(새로운 메모리 공간 확보 후 할당값 저장)

#### 값의 재할당

현재 변수에 저장된 값을 버리고 새로운 값을 저장하는 것
모든 변수가 선언시 undefined로 초기화 되기 때문에 변수에 처음 값을 할당하는 것도 재할당

- 재할당시 새로운 메모리 공간 확보 후 그 메모리 공간에 새로운 값 저장(이전 메모리에 덮어씌우기 X)
- 불필요한 값들은 가비지 콜렉터에 의해 메모리에서 자동 해제(언제 해제될지는 예측 불가능)
