### 10장 객체 리터럴

#### 객체란?

프로퍼티와 메서드로 구성된 집합체

- 프로퍼티: 객체의 상태를 나타내는 값
- 메서드: 프로퍼티를 참조하고 조작할 수 있는 동작

-> 객체는 상태와 동작을 하나의 단위로 구조화할 수 있다는 점에서 유용함

#### 객체지향 프로그래밍

객체의 집합으로 프로그램을 표현하려는 프로그래밍 패러다임

- 인스턴스: 클래스에 의해 메모리에 저장된 실체
- 클래스: 인스턴스를 만들기 위한 템플릿
- 객체: 클래스와 인스턴스를 포함한 개념

#### 객체 리처럴에 의한 객체 생성

중괄호를 이용해 객체를 생성하는 방법, 변수에 할당되는 시점에 js 엔진은 객체 리터럴을 해석해 객체를 생성

- 객체 리터럴은 값으로 표현되는 표현식, 객체 리터럴의 닫는 중괄호 뒤에는 세미콜론을 붙임
- 클래스를 정의하고 new 연산자와 함께 생성자를 호출할 필요 없음
- 객체 리터럴 외의 객체 생성 방식은 모두 함수를 사용해 객체를 생성

#### 프로퍼티

프로퍼티 키는 프로퍼티 값에 접근할 수 있는 이름으로서 식별자의 역할

- 빈 문자열 가능
- 식별자 네이밍 규칙 따를 필요 X, But 반드시 따옴표 사용
  -> 가급적이면 식별자 네이밍 규칙을 준수하는 프로퍼티 키를 사용할 것을 권장

- 프로퍼티 키애 문자열이나 심벌 값 외의 값을 사용하면 암묵적으로 문자열로 변환됨

#### 메서드

Js에서 함수는 값으로 취급할 수 있기 때문에 프로퍼티 값으로 사용가능
-> 메서드라고 명명(일반 함수와 구분짓기 위함)

#### 프로퍼티 접근

1. 마침표 프로퍼티 접근 연산자(.)을 이용한 마침표 표기법
2. 대괄호 프로퍼티 접근 연산자([])를 사용하는 대괄호 표기법
   - 따옴표로 감싼 문자열(숫자로 이루어진 문자열인 경우 따옴표 생략 가능)
   - 식별자
   - 식별자 네이밍 규칙을 따르지 않으면 반드시 대괄호 표기법으로 접근

주의

```js
var person={
    'last-name':'Lee',//식별자 네이밍 규칙을 따르지 않는 프로퍼티 키
}
person.'last-name' // SyntaxError
person.last-name
/**
 * person.last -> undfined
 * Node.js : name을 식별자로 인식,
 * name 찾을 수 없어서 Reference Error
 *
 * 브라우저 : person.last-(뺼셈연산) name(window 전역 객체의 name)
 * undfined - string
 * 결과: NaN */
```

#### ES6에서 추가된 객체 리터럴의 확장 기능

1. 프로퍼티 축약 표현

```js
var x = 1,
  y = 2;

//Before
var obj = {
  x: x,
  y: y,
};

//After
var obj = {
  x,
  y,
};
```

2. 객체 리터럴 내부에서도 계산된 프로퍼티 이름으로 프로퍼티 키 동적 생성 가능

```js
var prefix = "prop";
var i = 0;

//Before
var obj = {};
obj[prefix + "-" + i] = i;

//After
obj = {
  [prefix + "-" + i]: i,
};
```

3. 메서드 축약 표현

```js
var obj = {
  name: "Lee",
  //Before
  sayHi: function () {
    console.log("Hi!" + this.name);
  },
  //After
  sayHi() {
    console.log("Hi!" + this.name);
  },
};
```

주의: 메서드 축약 표현으로 정의한 메서드는 프로퍼티에 할당한 함수와 다르게 동작
