### 11장 원시 값과 객체의 비교

#### 원시값

- 값 자체 변경 불가능 **(불변성)**
- 확보해야할 메모리 공간의 크기를 사전에 정해둘 수 있음
- 원시값을 변수에 할당하면, 변수(할당된 메모리 공간)에는 **실제값**이 저장
- 원시값을 갖는 변수를 다른 변수에 할당하면 **원본의 원시값**이 복사되어 전달 **(값에 의한 전달)**
- 변수에 할당된 원시값을 변경하려면 재할당이 유일한 방법

#### 객체

- 값 자체 변경 가능
- 확보해야할 메모리 공간의 크기를 사전에 정해두지 못함
- 객체를 변수에 할당하면, 변수(할당된 메모리 공간)에는 **참조값**이 저장
- 객체를 가리키는 변수를 다른 변수에 할당하면 **원본의 참조값**이 복사되어 전달 **(참조에 의한 전달)**
- 변수에 할당된 객체는 직접 변경 가능

**-> 왜 객체는 변경가능한 값일까?**

- 객체를 생성하고 관리하는 일은 복잡하고 비용이 많이 듦
- 원시값처럼 값 자체를 복사해서 재할당하면 메모리의 효율적 소비가 어렵고 성능이 저하됨

장점 : 메모리를 효율적으로 사용 가능, 성능 향상
단점 : 여러개의 식별자가 하나의 객체를 공유할 수 있음

#### 얕은 복사, 깊은 복사

##### 객체에서의 의미

얕은 복사: 한 단계까지만 복사,
깊은 복사: 객체에 중첩되어 있는 객체까지 모두 복사

##### 값이 할당된 변수를 다른 변수에 할당할 때의 의미

얕은 복사: 객체를 할당한 변수를 다른 변수에 할당
깊은 복사: 원시값을 할당한 변수를 다른 변수에 할당

주의: 자바스크립트에서는 결국 변수에 원시값/객체가 저장된 메모리 주소가 저장되어, **값에 의한 전달만이 존재**
전달되는 값의 종류가 원시값/객체에 따라 동작 방식이 다름
